# lyra-clean-code

Twenty clean-code rules for code that is cheap to change and easy to test, each with a bad/good example in a real language. The rules span naming, functions, SOLID, component cohesion, error handling, testability, and state. Each one ties back to testability — if a function resists a test, a rule is being broken somewhere. Cite rule numbers in review instead of taste.

**Reach for it when:**

- Writing, reviewing, or refactoring code in TS, Python, PHP, Go, Rust, Java, C#, or Ruby.
- You want a concrete rule to cite instead of "this feels wrong."

**Don't:**

- Expect framework-specific guidance — examples are language-agnostic.
- Use it for stylistic debates that don't touch correctness or testability.
